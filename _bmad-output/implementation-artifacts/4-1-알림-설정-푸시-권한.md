## Story

As a 엔드유저,
I want 푸시 알림을 ON/OFF로 토글하고 알림 시간(기본 저녁 8시 KST)을 변경하며, 권한 거부 시에도 빈 화면 대신 *"설정에서 알림 허용 시 미기록 nudge가 동작합니다"* 안내 + 설정 deep link를 받기를,
So that 내 라이프스타일에 맞는 nudge 정책을 직접 결정하고, 권한 거부 후에도 막다른 UX 없이 설정 화면으로 명확히 진입할 수 있다.

## Acceptance Criteria

1. **AC1 — 알림 설정 4컬럼 마이그레이션 (`users` 테이블 0013)**
   **Given** Story 1.5에서 7컬럼 + ENUM 2종이 박힌 `users` 테이블, **When** Alembic `0013_users_notification_settings` upgrade 실행, **Then** 4컬럼 추가:
   - `notifications_enabled` `BOOLEAN NOT NULL DEFAULT true` — *기본 ON*. PIPA Art.22(3) marketing 동의는 별 게이트지만, 본 nudge는 *서비스 이행*(미기록 알림은 식단 관리 서비스 본연 기능 — 마케팅 메시지 X) → opt-out 모델 정합. 사용자 ON/OFF 토글로 즉시 차단 가능.
   - `notification_time` `TIME NOT NULL DEFAULT '20:00:00'` — KST 기준 *알림 시간*(서버 cron이 `notification_time + 30분` 시점에 미기록 push 발송. 본 스토리는 컬럼만 박고 Story 4.2가 cron 소비).
   - `notification_timezone` `TEXT NOT NULL DEFAULT 'Asia/Seoul'` — IANA tz 식별자. 컬럼만 박고 Story 4.2가 APScheduler `timezone='Asia/Seoul'` SOT로 사용. 변경 UI는 본 스토리 OUT(MVP는 한국 사용자 한정 — Story 8.4 polish forward).
   - `expo_push_token` `TEXT NULL` — Expo push token (`ExponentPushToken[xxx]` 형식). 단일 device만 보유(MVP 단순화 — multi-device support는 Story 8.4 polish forward).
   - **CHECK 제약**: `ck_users_notification_time_kst_format` `notification_time IS NULL OR notification_time::text ~ '^[0-2][0-9]:[0-5][0-9]:00$'` — 분 단위 정렬 강제(초 자유 입력 차단, APScheduler 일별 잡 안정성).
   - **partial UNIQUE index**: `ux_users_expo_push_token_active` `ON users (expo_push_token) WHERE expo_push_token IS NOT NULL AND deleted_at IS NULL` — 동일 device가 여러 user에 매핑되는 회귀 차단(예: 사용자 A 로그아웃 후 동일 device에서 사용자 B 로그인 시 B의 token이 A를 unset).
   - downgrade는 컬럼 4 + index drop 역순. 기본값 부여 시 lock window는 ms 수준(8주 MVP user count <100 — 백필 SOP 무필요).

   **Given** `api/app/services/notification_service.py` 신규, **When** 4 helper 함수 export, **Then**:
