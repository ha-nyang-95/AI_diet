## Story

As a **외주 클라이언트 개발자(인수인계 대상)**,
I want **git clone 후 `docker compose up` 1-cmd로 풀스택 환경(Postgres+pgvector+Redis+FastAPI+seed)이 부팅되고 PR 시 ruff·mypy·pytest·tsc·openapi diff 게이트가 자동 작동하기를**,
So that **README + 8시간 내 로컬 환경 구축 + 자체 데이터 통합이 가능한 인수인계 가능 산출물의 토대가 박힌다.**

## Acceptance Criteria

> **BDD 형식 — 각 AC는 독립적 검증 가능. 모든 AC 통과 후 `Status: ready-for-review`로 전환.**

1. **AC1 — Docker 1-cmd 부팅**
   **Given** 빈 디렉토리 + 본 repo clone, **When** `docker compose up`,
   **Then** `postgres`(pgvector pg17) + `redis` + `api`(FastAPI) + `seed`(one-shot) 컨테이너가 일괄 부팅되고 `api` 컨테이너 healthcheck 통과까지 ≤ 10분.
   - `api`는 `postgres`/`redis` healthy 후에만 시작 (depends_on `condition: service_healthy`).
   - `seed` 컨테이너는 `postgres` healthy 후 1회 실행 후 종료(`restart: no`).

2. **AC2 — Web 부트스트랩 (Next.js 16.2.4)**
   `pnpm dev` 실행 시 `localhost:3000` 응답. App Router + Tailwind + TS + Turbopack + `src/` + import alias `@/*` 모두 starter 디폴트로 적용.
   - `web/AGENTS.md` 존재 (starter가 자동 생성하면 그대로 보강, 아니면 수동 작성 — Task 2.2).

3. **AC3 — Mobile 부트스트랩 (Expo SDK 54)**
   `pnpm start`(또는 `npx expo start`) 시 Expo dev menu가 동작하고 빈 라우팅 화면(Expo Router file-based, `app/index.tsx`) 응답.

4. **AC4 — API 헬스체크 + Swagger**
   `localhost:8000`에서 다음 엔드포인트 모두 200 응답:
   - `GET /healthz` — **liveness only** (프로세스 생존 신호만, 외부 의존성 무관 — k8s liveness 컨벤션 준수). 응답 body: `{"status": "ok"}`. DB·Redis 일시 장애가 컨테이너 재시작 루프를 트리거하지 않도록 의존성 분리.
   - `GET /readyz` — readiness, Postgres + Redis + pgvector extension 존재 확인 (`SELECT extname FROM pg_extension WHERE extname='vector'`). 모든 자원 ok 시 200, 하나라도 실패 시 503 (k8s readiness 컨벤션).
   - `GET /docs` — Swagger UI (FastAPI 자동 생성)
   - `GET /openapi.json` — OpenAPI schema

   > **AC4 amended (code review, 2026-04-27)**: 원본 AC4는 `/healthz`에서 DB ping을 명시했으나, k8s liveness 컨벤션상 외부 의존성과 분리되어야 함(DB 일시 장애 → 재시작 루프 위험). DB ping은 `/readyz`에 통합. Spec·코드 동기화 완료.

5. **AC5 — 관측성 + Rate limit 골격**
   다음 SDK·미들웨어가 `/api/app/main.py` 부팅 시 초기화 완료(설치만, 정교화는 Epic 8):
   - `sentry-sdk`(Python) — DSN env 설정 시 활성, `before_send` hook이 `app/core/sentry.py`에 등록되어 민감정보 마스킹 자리표시자(`mask_event(event)`) 동작.
   - `structlog` — JSON renderer + `mask_processor`(자리표시자) 등록 + 표준 필드(`timestamp`, `level`, `request_id`, `event`).
   - `slowapi` + Redis backend — 분당 60 / 시간당 1000 / LangGraph 분당 10 3-tier 키 정의 1개 등록(라우터 적용은 후속 스토리).
   - `request_id` 미들웨어 — UUIDv4 생성 + 응답 헤더 `X-Request-ID`.

6. **AC6 — 환경 변수 + secret 표준화**
   - 루트 `.env.example`에 모든 필수 환경변수 키 + 1줄 설명 + 권장 값(또는 placeholder) 기재.
   - `.env`, `.env.local`은 `.gitignore`에 등재.
   - 최소 변수: `DATABASE_URL`, `REDIS_URL`, `JWT_USER_SECRET`, `JWT_ADMIN_SECRET`, `JWT_USER_ISSUER`, `JWT_ADMIN_ISSUER`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `LANGSMITH_API_KEY`, `LANGSMITH_PROJECT`, `LANGCHAIN_TRACING_V2`, `SENTRY_DSN`, `GOOGLE_OAUTH_CLIENT_ID`, `GOOGLE_OAUTH_CLIENT_SECRET`, `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET`, `R2_PUBLIC_BASE_URL`, `EXPO_ACCESS_TOKEN`, `MFDS_OPENAPI_KEY`, `TOSS_SECRET_KEY`(또는 `STRIPE_SECRET_KEY`).

## Dev Notes

### 핵심 Architecture 결정 (반드시 따를 것)

**Repo 구조 — 단일 repo + multi-folder, 워크스페이스 X**
[Source: architecture.md#Repo 구조 결정]
- pnpm workspace / Turborepo / Nx **모두 금지**. Python(api) ↔ TS(mobile/web) 워크스페이스 통합 불가 — 반쪽짜리 monorepo는 안티패턴.
- 컴포넌트 경계 = 폴더 = 컨테이너 (Docker Compose 1-cmd와 정합).
- TS 타입 공유는 OpenAPI 단일 출처 → `openapi-typescript` 자동 생성 (`pnpm gen:api`).

**버전 핀 (절대 임의 변경 금지)**
[Source: architecture.md#Component-별 Starter, #Data Architecture]
- Postgres **17** + `pgvector/pgvector:pg17` 이미지 + pgvector **0.8.2**
  - ⚠ architecture.md 본문 일부에 `pg16`이 한 곳 남아있으나(line 216), 데이터 결정 표(line 285)와 epics.md AC가 모두 **pg17**로 일치 — pg17 사용.
- Python **3.13**, FastAPI(standard), LangGraph **1.1.9**, SQLAlchemy **2.0.49** async + asyncpg
- Next.js **16.2.4** (App Router + Tailwind v3 + Turbopack)
- Expo **SDK 54**
- 패키지 매니저: **uv**(Python), **pnpm**(TS) — pip/poetry/npm/yarn 금지

**`lg_checkpoints` 스키마는 Alembic 외부**
[Source: architecture.md#Decision Impact Analysis "Cross-Component Dependencies"]
- LangGraph `AsyncPostgresSaver`는 자체 `.setup()`으로 스키마 관리 — Alembic이 추적 X.
- 본 스토리에서는 `01-pgvector.sql`에 `CREATE SCHEMA IF NOT EXISTS lg_checkpoints;`만 박고, 테이블 생성은 Story 3.3에서 `AsyncPostgresSaver(...).setup()` 호출로 처리.

**API 응답 표준 (전체 횡단)**
[Source: architecture.md#API & Communication Patterns, #Format Patterns]
- 성공: 직접 객체 (wrapper 금지). `{"id": ..., ...}`
- 에러: **RFC 7807 Problem Details** — `application/problem+json` + `type/title/status/detail/instance` + 우리 확장 `code`
- 응답·요청 wire format **snake_case 양방향**. Pydantic `alias_generator=to_camel` 도입 금지.
- URL: REST kebab-plural noun + `/v1/` (예: `/v1/audit-logs`). 본 스토리는 health 엔드포인트만이라 `/healthz`, `/readyz`는 예외 (운영 표준).

### 명명 규칙 (전체 횡단)

[Source: architecture.md#Naming Patterns]
- DB: snake_case + 복수 (`users`, `meals`). FK: `<singular>_id`. 인덱스: `idx_<table>_<cols>`. Enum: `<table>_<col>_enum`.
- API URL: REST kebab-plural + `/v1/` (예: `GET /v1/audit-logs`). Query: snake_case.
- JSON wire: **snake_case 양방향** (Pydantic alias 금지).
- Python: snake_case / PascalCase / UPPER_SNAKE.
- TS: camelCase / PascalCase / UPPER_SNAKE.
- 파일명: Python `.py` snake_case / React `.tsx` PascalCase / 기타 TS kebab-case / Next.js·Expo Router 폴더 kebab-case.

### CI 게이트 세부 (`.github/workflows/ci.yml`)

[Source: architecture.md#Infrastructure & Deployment "배포 게이트", epics.md#Additional Requirements "CI Gate"]
- 트리거: `pull_request` + `push` to main
- 잡 7종(AC7 매핑):
  1. `lint-format-api` — `ruff check api/` + `ruff format --check api/`
  2. `typecheck-api` — `mypy api/app`
  3. `test-api` — postgres + redis service container, `uv run pytest --cov`, coverage ≥ 70% 미달 시 fail
  4. `typecheck-web` — `pnpm --filter web tsc --noEmit` (단 워크스페이스 X이므로 `cd web && pnpm tsc --noEmit`)
  5. `typecheck-mobile` — `cd mobile && pnpm tsc --noEmit`
  6. `openapi-diff` — `test-api` 잡에서 OpenAPI schema dump → 기존 `web/src/lib/api-client.ts`와 git diff 비교, 차이 발생 시 fail
  7. `alembic-dry-run` — `cd api && uv run alembic upgrade head --sql` 결과를 PR 코멘트로 게시 (실패 시 fail, 출력은 정보 게시)

- 모든 잡 fail 시 머지 차단 (`branch protection rule`은 README에 SOP 명시 — 본 스토리는 *워크플로우 작성*까지, GitHub UI 설정은 SOP에 위임).
- `pre-commit` hook은 본 스토리 **OUT** (Epic 8에서 도입 가능) — CI에서 lint 게이트로 충분.
