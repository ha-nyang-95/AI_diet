## Story

As a **엔드유저**,
I want **첫 진입 시 디스클레이머 + 약관 동의 + 개인정보 동의(민감정보 별도)를 통과하고 앱·웹 어디서나 다시 열람하기를**,
So that **의료기기 미분류 + PIPA 민감정보 처리 의무를 충족한 상태로 핵심 기능에 진입하고 권리 행사 시 텍스트를 재확인할 수 있다.**

부수적으로 본 스토리는 Epic 1-7 전체 횡단의 동의 인프라를 박는다 — `consents` 테이블, `app/api/deps.py:require_basic_consents` Depends, `legal` 정적 텍스트 SOT(한·영). Story 1.4(자동화 의사결정 동의)가 같은 테이블에 컬럼 2개를 추가한다. Story 1.5(건강 프로필) / Story 2.1+(식단)은 본 스토리의 dependency를 사용해 미동의 사용자 진입을 차단한다. Epic 3(분석 카드)은 본 스토리의 `DisclaimerFooter` 컴포넌트를 임포트한다.

3. **AC3 — 동의 통과 시 `consents` 테이블에 항목·시점·버전 기록**
   **Given** 4종 모두 체크된 사용자, **When** 모바일/Web이 `POST /v1/users/me/consents` 호출(body: `{disclaimer_version, terms_version, privacy_version, sensitive_personal_info_version}` — 각 버전 string은 `GET /v1/legal/*` 응답의 `version` 필드와 동일), **Then**:
     - 백엔드가 `consents` 테이블에 user 기준 row UPSERT(이미 있으면 갱신):
       - `disclaimer_acknowledged_at = now()`, `terms_consent_at = now()`, `privacy_consent_at = now()`, `sensitive_personal_info_consent_at = now()`
       - `disclaimer_version`, `terms_version`, `privacy_version`, `sensitive_personal_info_version` (string, 예: `ko-2026-04-28`)
       - `ip_address` (`request.client.host`, `inet`), `user_agent` (`request.headers.get("User-Agent")` truncate 512자) — 감사 흔적
     - 응답 status 201(신규) 또는 200(갱신), body `{disclaimer_acknowledged_at, terms_consent_at, privacy_consent_at, sensitive_personal_info_consent_at, basic_consents_complete: true}`.

     - 하나라도 누락되면 **400** RFC 7807 + `code=validation.error` (`api/app/main.py`의 `validation_exception_handler`가 Pydantic 422 → 400으로 매핑 — Story 1.2 정합).

    - `main.py`: `app.include_router(legal_router, prefix="/v1/legal", tags=["legal"])`.
    - `app/domain/legal_documents.py` 신규 — SOT:
      ```python
      LegalDocType = Literal["disclaimer", "terms", "privacy"]
      LegalLang = Literal["ko", "en"]

      @dataclass(frozen=True)
      class LegalDocument:
          type: LegalDocType
          lang: LegalLang
          version: str            # 예: "ko-2026-04-28"
          title: str
          body: str               # 풀텍스트(여러 문단 \n\n 구분)
          updated_at: datetime    # ISO 8601 UTC

      LEGAL_DOCUMENTS: dict[tuple[LegalDocType, LegalLang], LegalDocument] = {
          ("disclaimer", "ko"): LegalDocument(type="disclaimer", lang="ko", version="ko-2026-04-28", title="면책 조항", body=DISCLAIMER_KO_TEXT, updated_at=datetime(2026, 4, 28, tzinfo=timezone.utc)),
          ("disclaimer", "en"): LegalDocument(...),
          ("terms",      "ko"): ...,
          ("terms",      "en"): ...,
          ("privacy",    "ko"): ...,
      ```

### Architecture decisions (반드시 따를 것)

**동의 게이트 분리 (절대 통합 금지)**

- **인증 게이트** (`current_user` Depends, Story 1.2): JWT 검증 + DB user lookup. *모든* 보호 라우트.
- **기본 동의 게이트** (`require_basic_consents` Depends, **본 스토리**): 4종 동의(disclaimer/terms/privacy/sensitive_personal_info) 통과 + 현재 SOT 버전 일치. *핵심 사용자 데이터 작성·조회* 라우트(Story 1.5 `POST /users/me/profile`, Story 2.1+ `/v1/meals/*`).
- **자동화 의사결정 게이트** (`require_automated_decision_consent` Depends, Story 1.4): PIPA 2026.03.15 별도 동의. *분석 호출* 라우트만(`/v1/analysis/stream`).
- **이유**: (a) `/v1/users/me`는 *동의 미통과* 사용자도 호출 가능해야(클라이언트가 자기 정보 fetch) — `current_user`만 Depends. (b) 분석 호출은 LLM 비용 0 보호 위해 *자동화 의사결정 동의* 추가 게이트 필요 — Story 1.4가 별 dependency.

**SOT 위치 (텍스트 콘텐츠)**

- 디스클레이머/약관/개인정보 처리방침 텍스트의 SOT는 `app/domain/legal_documents.py` 모듈 const. CMS·DB row·외부 파일 X — 1인 8주 정합.
- Web RSC는 backend `/v1/legal/{type}?lang=...` 호출 + 24h ISR 캐시(`fetch(..., { next: { revalidate: 86400 }})`). 모바일은 `useQuery` `staleTime: 24h`.

**`consents` 테이블 — 컬럼 per 동의 타입 (event-style 미채택)**

- Option A(채택): 1 user 1 row, 컬럼 per 동의(`*_at` + `*_version`). Story 1.4가 같은 row에 컬럼 2-3개 추가.
- Option B(기각): append-only `(user_id, consent_type, granted_at, revoked_at, version)` rows. 감사 history 풍부하나 8주 MVP에 ROI 낮음 + Story 1.4 AC4 텍스트(`consents.automated_decision_consent_at`)가 컬럼 명을 직접 참조 → Option A 채택.

### Anti-pattern 카탈로그 (본 스토리 영역)

[Source: architecture.md#Anti-pattern 카탈로그 (line 557-565), Story 1.2 Anti-pattern 카탈로그]

- ❌ "전체 동의" 단일 체크 박스로 4종 묶기 — PIPA 22조 별도 동의 의무 위반.
- ❌ 민감정보 동의를 일반 개인정보 동의에 합치기 — PIPA 23조 별도 동의 의무 위반.
- ❌ 텍스트 콘텐츠를 DB row로 보관(MVP 단계) — `app/domain/legal_documents.py` 모듈 const SOT.
- ❌ 약관 풀텍스트를 모바일·web 양쪽에 중복 임베드 — backend `/v1/legal/*` SOT.
- ❌ `current_user` Depends에서 동의 체크 강제 차단 — `require_basic_consents` 별 dependency로 분리.
- ❌ Web onboarding UI 미제공(redirect만) — 외주 영업 데모 시 Web만으로 시연 시 dead-end. 본 스토리는 *minimal 4-체크박스 UI*로 lock-in 회피.
- ❌ `consents` 테이블에 Story 1.4 자동화 의사결정 컬럼 미리 박기 — scope creep, Story 1.2 패턴 정합.
- ❌ Pydantic `alias_generator=to_camel`로 wire format 변경 — snake_case 양방향 정합.
- ❌ `raise HTTPException(403, "...")` 직접 사용 — `BasicConsentMissingError` / `ConsentVersionMismatchError` 도메인 예외.
- ❌ `legal` 라우터에 `current_user` Depends 추가 — public endpoint 정합.
- ❌ `consent.version_mismatch` status 422 — Pydantic ValidationError 422와 충돌. **409 Conflict** 사용.
- ❌ `users.updated_at` 명시 set — `onupdate=text("now()")` 자동 갱신 신뢰(Story 1.2 P8 patch 정합).
- ❌ `legal` 라우터 클라이언트 호출 시 cookie 동봉 — public endpoint, `headers: {}` 명시(`fetchLegalDocument`).

### Previous Story Intelligence (Story 1.2 학습)

**적용 패턴(반드시 재사용):**

- **UPSERT race 방지**: Story 1.2 P11 패턴(`IntegrityError` catch + rollback + 재조회) 또는 `INSERT ... ON CONFLICT DO UPDATE`. 본 스토리는 후자 채택(`pg_insert(...).on_conflict_do_update(...)`) — 단일 statement 원자성.

- **테스트 fixture 패턴**: `tests/conftest.py`의 `_truncate_user_tables` *before yield* (Story 1.2 T1 patch). `consents` 테이블 추가 시 동일 패턴.
- **assertion 정확 매칭**: `code.startswith("consent.")` → 구체 코드(`consent.basic.missing` / `consent.version_mismatch`) — Story 1.2 T11 patch 정합.
